# Coding_Rule
收集了编程时应该遵循的规则，有统一的规则可以让代码更加工整

***
# 1 概述
## 1.1 编写目的
为规范开发人员的代码编写的参考依据和统一标准。
## 1.2 适用范围
本文档适用于所有开发人员。

# 2 命名规范
## 2.1 常量命名
&emsp;**常量的命名应使用*全大写与下划线*的组合命名**

``` C++
#define MIN_SIZE 50
const int MAX_SIZE = 100;
```

## 2.2 变量的命名 
### 2.2.1 局部变量
&emsp;**局部变量应使用 *小驼峰* 的格式命名**

``` C++
int mapSize = 70;
```

### 2.2.2 私有变量
&emsp;**私有变量应使用 *下划线开头+小驼峰* 的格式命名**

``` C++
private int _nowPosition = 0;
```

### 2.2.3 全局变量
&emsp;**全局变量应使用 *小驼峰+下划线结尾* 的格式命名**

``` c++
int nowPosition_ = 0;
```

## 2.3 函数命名 
&emsp;**一般函数用 *全小写与下划线* 的组合命名**

``` C++
int flush_map(char *, int );
```

## 2.3 对象的命名
&emsp;**对象应使用 *大驼峰* 的格式命名**



# 3 排版要求
## 3.1 程序块缩进
&emsp;**程序要采用缩进风格编写,缩进的空格数为4个。** <br/>

## 3.2 程序块之间空行
&emsp;**相对独立的程序块之间、变量说明之后必须空行。** <br/>
&emsp;示例 ： 如下例子不符合规范。

``` C++
if( !flag ) {
    ... // program code 
}
flag += 2 ;
``` 

&emsp;应如下书写

``` C++
if( !flag ) {
    ... // program code 
}

flag += 2 ;
``` 

## 3.3 长语句与长表达式
&emsp;**较长的语句（>80字符）要分成多行书写，长表达式要在低优先级操作符处划分新行，操作符放在新行之首，划分出的新行要进行适当的缩进，使排版整齐，语句可读。** <br/>
&emsp; 示例 ：
``` C++
perm_count_msg.head.len = NO7_TO_STAT_PERM_COUNT_LEN
                        + STAT_SIZE_PER_FRAM * sizeof( _UL );
```

## 3.4 条件、循环语句
&emsp;**if、for、do、while、case、switch、default等语句自占一行，且if、for、do、while等语句的执行语句部分无论多少都要加括号{}。** <br/>
&emsp;示例：如下例子不符合规范。
```C++
if (pUserCR == NULL) return;
```
应如下书写：
```C++
if (pUserCR == NULL){
    return;
}
```

## 3.5 程序块分界符
&emsp;**程序块的分界符（如C/C++语言的大括号‘{’和‘}’）应各独占一行并且位于同一列 或者 位于同一行前加空格，同时与引用它们的语句左对齐。在函数体的开始、类的定义、结构的定义、枚举的定义以及if、for、do、while、switch、case语句中的程序都要采用如下的缩进方式。** <br/>
&emsp;示例：如下例子不符合规范。
``` C++
for(...){
    ... // program code
}
if (...)
    {
    ... // program code
    }
void example_fun( void )
{
    ... // program code
    }
... // program code
```
应如下书写。
``` C++
for (...) {
    ... // program code
}
for (...) 
{
    ... // program code
}
```

## 3.6 操作符前后空格
&emsp;**在两个以上的关键字、变量、常量进行对等操作时，它们之间的操作符之前、之后或者前后要加空格；进行非对等操作时，如果是关系密切的立即操作符（如－>），后不应加空格。** <br/>
&emsp;说明：采用这种松散方式编写代码的目的是使代码更加清晰。
由于留空格所产生的清晰性是相对的，所以，在已经非常清晰的语句中没有必要再留空格，如果语句
已足够清晰则括号内侧 即左括号后面和右括号前面 不需要加空格，多重括号间不必加空格，因为
在 C/C++ 语言中括号已经是最清晰的标志了。
在长语句中，如果需要加的空格非常多，那么应该保持整体清晰，而在局部不加空格。给操作符留空
格时不要连续留两个以上空格。 <br/>
示例： <br/>
(1)
逗号、分号只在后面 加空格。
```C++
int a, b, c;
```
(2)比较操作符 , 赋值操作符 "="、 "+="，算术操作符 "+" 、 "% "，逻辑操作符 "&" 、 "&&"，位域
操作符 "<<" 、 "^" 等双目操作符的前后加空格。
``` C++
if (current_time >= MAX_TIME_VALUE)
a = b + c;
a *= 2;
```

## 3.7 其他
&emsp;**一行程序以小于 80 字符为宜，不要写得过长。**

# 4 注释
## 4.1 有效注释量
&emsp;**<font color="Red">一般情况下，源程序有效注释量必须在 20 ％以上。** <br/>
&emsp;说明：注释的原则是有助于对程序的阅读理解，在该加的地方都加了，注释不宜太多也不
能太少，注释语言必须准确、易懂、简洁。</font>
